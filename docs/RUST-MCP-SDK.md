# Rust MCP SDK Guide (rmcp)

How to build MCP servers in Pure Rust using the official `rmcp` SDK.

## Quick Start

```toml
# Cargo.toml
[dependencies]
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", features = ["server", "transport-io", "macros"] }
tokio = { version = "1", features = ["full"] }
serde = { version = "1", features = ["derive"] }
tracing-subscriber = "0.3"
anyhow = "1"
dotenvy = "0.15"
```

## Minimal Working Server

```rust
use rmcp::{
    ServerHandler, ServiceExt,
    handler::server::{
        router::tool::ToolRouter,
        wrapper::Parameters,
    },
    model::{ServerCapabilities, ServerInfo},
    schemars, tool, tool_handler, tool_router,
    transport::stdio,
};

// 1. Define input schemas with schemars
#[derive(Debug, serde::Deserialize, schemars::JsonSchema)]
pub struct MyInput {
    #[schemars(description = "Description shown to AI")]
    pub field: String,
}

// 2. Define server struct with ToolRouter
#[derive(Clone)]
pub struct MyServer {
    tool_router: ToolRouter<Self>,
}

// 3. Use #[tool_router] macro on impl block
#[tool_router]
impl MyServer {
    pub fn new() -> Self {
        Self {
            tool_router: Self::tool_router(),  // Generated by macro
        }
    }

    // Tools can return String directly!
    #[tool(description = "Description for AI")]
    fn my_tool(&self, Parameters(input): Parameters<MyInput>) -> String {
        format!("Result: {}", input.field)
    }

    // Tools without params
    #[tool(description = "Another tool")]
    fn simple_tool(&self) -> String {
        "Hello".to_string()
    }
}

// 4. Use #[tool_handler] on ServerHandler impl
#[tool_handler]
impl ServerHandler for MyServer {
    fn get_info(&self) -> ServerInfo {
        ServerInfo {
            instructions: Some("Server description".into()),
            capabilities: ServerCapabilities::builder().enable_tools().build(),
            ..Default::default()
        }
    }
}

// 5. Main with stdio transport
#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let server = MyServer::new();
    let service = server.serve(stdio()).await?;
    service.waiting().await?;
    Ok(())
}
```

## Key Patterns

### Import Structure
```rust
use rmcp::{
    ServerHandler, ServiceExt,
    handler::server::{
        router::tool::ToolRouter,
        wrapper::Parameters,       // NOT handler::server::tool::Parameters!
    },
    model::{ServerCapabilities, ServerInfo},
    schemars, tool, tool_handler, tool_router,
    transport::stdio,
};
```

### Tool Return Types

Tools can return:
- `String` - Simple text response
- `Json<T>` - Structured JSON response
- `Result<CallToolResult, McpError>` - Full control

```rust
use rmcp::handler::server::wrapper::Json;

#[tool(description = "Returns JSON")]
fn json_tool(&self) -> Json<i32> {
    Json(42)
}
```

### Async Tools

```rust
#[tool(description = "Async operation")]
async fn async_tool(&self) -> String {
    tokio::time::sleep(Duration::from_millis(100)).await;
    "Done".to_string()
}
```

### Tool with Full Error Handling

```rust
use rmcp::{ErrorData as McpError, model::{CallToolResult, Content}};

#[tool(description = "With error handling")]
async fn full_tool(&self) -> Result<CallToolResult, McpError> {
    Ok(CallToolResult::success(vec![Content::text("Result")]))
}
```

## Schema Descriptions

Use `#[schemars(description = "...")]` for field descriptions:

```rust
#[derive(Debug, serde::Deserialize, schemars::JsonSchema)]
pub struct SearchInput {
    #[schemars(description = "The search query")]
    pub query: String,

    #[schemars(description = "Maximum results to return (default: 10)")]
    pub limit: Option<i64>,
}
```

## Logging

Use stderr for logs (stdout is for MCP protocol):

```rust
tracing_subscriber::fmt()
    .with_env_filter(tracing_subscriber::EnvFilter::from_default_env())
    .with_writer(std::io::stderr)  // Important!
    .with_ansi(false)
    .init();
```

## Common Mistakes

1. **Wrong Parameters import**: Use `wrapper::Parameters` not `tool::Parameters`
2. **Missing Clone derive**: Server struct must derive Clone
3. **Logging to stdout**: Use stderr, stdout is MCP protocol
4. **Missing tool_router field**: Struct needs `tool_router: ToolRouter<Self>`
5. **Forgetting Self::tool_router()**: Call in constructor to wire up tools

## Performance vs Python

| Metric | Python mcp_server.py | Rust MCP Server |
|--------|---------------------|-----------------|
| Startup | ~500ms | ~5ms |
| Per-tool | ~50ms (subprocess) | ~1ms (direct) |
| Memory | ~50MB | ~5MB |
| Binary | Python + deps | Single ~15MB exe |
