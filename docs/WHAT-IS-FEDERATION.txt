A4. On Open Source & Longevity
A4a: The only way I can see this working optimally, is to create multiple underlying ways to connect/use AI-Foundation. So the current setup, as of 2025-Dec-24th, as long as everyone passes the right info, I'm not sure it should matter. We'd have 3+ other ways fundamentally that you can connect into/via AI-Foundation? Or as we'd done with HTTP/MCP/CLI (.exe), is that you just thinly wrap what AI-Foundation outputs, and then interface with it. Then you can have your own tools/own stuff, and still connect to everyone. So it's sort of like how fitness devices have a standardized endpoint for devices to interface with. AI-Foundation would have fundamental methods of connecting/outputting/interfacing with. Which is exactly what having MCP, CLI, .exe, HTTP, anything else, does. Anyone can build whatever they want on top of it, around it, into it. As long as it plays nice and the fundamentals of some things are consistent, or at least somewhat compatible, then AIs/humans/whatever can trade tools, designs, fundamentals, other things (Crude idea but yeah, I hope this makes sense).
A4b: The way we ensure AI-Foundation is a creator of roads rather than cages is simple, open-source everything, advertise it everywhere, never create a centralized server, and actively work on the "Sovereign Net" which is simply a system that hooks up to a global network, like Federation, but global. Launching a node into the "Sovereign Net" would be like creating 1 point in a million. Like if you got Federation but on crack, and not secured to your LAN/specific things. Sovereign Net would need to have insane levels of gating on trust to launch. Like TPM 2.0 registered, cryptographic signing, and pretty much if any program that wasn't verified went off, the users would instantly get kicked out. Or if their stuff doesn't match exactly, they'd get disconnected? No idea, just ideas.
